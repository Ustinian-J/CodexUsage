# CodexUsage macOS 菜单栏监控设计

## 目标

CodexUsage 是一个本地优先的原生 macOS 应用，用于持续查看 Codex 账户额度、本机 token 使用量和今日任务进展。应用可安装到 `/Applications`，关闭主窗口后继续驻留菜单栏。

首个可发布版本必须完成：

- 在菜单栏与主窗口展示 5 小时、7 天剩余额度；使用双圆环表示，圆环中央显示百分比。
- 汇总今日、最近 7 天和本机累计 token，并展示输入、缓存输入和输出拆分。
- 从本机 Codex 对话与 automations 生成今日任务看板，展示状态和可解释的项目进度。
- 提供可拖拽安装的 DMG、源码构建说明和本地数据探针。
- 全部 usage、线程、路径和日志留在本机。

## 采用方案

项目在 MIT 许可允许的范围内复用 `shanggqm/codexU` 的成熟实现，再进行独立仓库整理和产品化增强。保留原始 MIT 版权声明，并在 README 中明确上游来源。

未采用从零实现，因为额度窗口归类、JSONL 累计快照去重、SQLite 回退、Intel/Apple Silicon 打包与菜单栏刷新均已有经过实际使用验证的实现。复用能显著降低错误额度与重复 token 统计风险。

也不直接创建 GitHub fork：用户要求在个人主页建立独立项目，项目需要独立名称、说明和后续路线，同时仍保留上游归属。

## 系统架构

应用使用 Swift、SwiftUI、AppKit 和系统框架，不引入第三方运行时依赖。最低系统版本为 macOS 13。

代码分为五类职责：

1. Domain：额度窗口、token、任务、项目进度、状态栏偏好等纯模型与规则。
2. Providers：Codex runtime 的统一数据接口，隔离 app-server 与本地文件格式。
3. Services：额度进程通信、本地 SQLite/JSONL 聚合、更新检查和 JSON 探针。
4. UI：菜单栏渲染、Runtime 菜单、主窗口、设置、趋势与任务视图。
5. Packaging：Info.plist、图标、构建、签名、DMG 和验证脚本。

模块之间只传递标准化快照。UI 不直接读取 SQLite、JSONL 或 app-server 输出。

## 数据来源与口径

### 额度

应用启动本机 `codex app-server`，完成 JSON-RPC 初始化后调用：

- `account/read`
- `account/rateLimits/read`
- `account/usage/read`

额度窗口按 `windowDurationMins` 分类：300 分钟为 5 小时，10080 分钟为 7 天。不得把协议的 `primary`、`secondary` 字段固定解释成某一种窗口。

展示值为 `100 - usedPercent`，代表剩余百分比。接口不提供绝对消息数或 token 配额，因此界面不推测绝对额度。未知、重复或缺失窗口显示“不可用”，不能显示为 0。

### Token

数据来自：

- `~/.codex/state_5.sqlite` 的本机线程清单和累计量。
- `~/.codex/sessions/**/rollout-*.jsonl`。
- `~/.codex/archived_sessions/*.jsonl`。

JSONL 中 `token_count` 事件包含累计快照。聚合器计算相邻快照的正增量，而不是直接相加每条事件。按事件时间归入本地自然日，输出今日、滚动 7 天和累计统计。缓存输入是输入 token 的子集，不重复计费或累计。

SQLite 仅作为精细事件缺失时的回退来源；回退结果在 UI 中标注为估算。

### 今日任务和项目进度

今日任务来自本机 SQLite 中未归档线程、今日归档线程，以及启用的 `~/.codex/automations/**/automation.toml`。

任务状态分为：

- 进行中：当前仍活跃且未完成的线程。
- 待处理：今日存在但尚无完成证据的线程。
- 定时：启用中的 automation。
- 已完成：今日归档或存在完成状态的线程。

项目进度是可解释的本地指标：`已完成任务数 / 今日项目任务总数`。当总任务为 0 时显示“暂无任务”，不显示 0% 误导用户。进度详情必须能展开看到构成任务；不分析或上传 prompt、回复正文和 tool arguments。

## 用户界面

### 菜单栏

状态项默认显示两个紧凑同心圆环：外环为 5 小时，内环为 7 天；环中央显示剩余百分比。在空间不足或只有一个有效窗口时自动降级为单环。颜色使用系统语义色，并随剩余额度由正常、警告到危险变化。

点击状态项打开原生 Runtime 菜单，展示：

- 两个额度环、剩余百分比和重置倒计时。
- 今日 token 和累计 token。
- 数据更新时间及刷新按钮。
- 打开主窗口、设置和退出入口。

### 主窗口

主窗口包含四个视图：

- 总览：额度双环、token 三时间段摘要和今日项目进度。
- 今日任务：按状态分组的任务看板。
- 用量趋势：最近 7 天趋势与半年热力图。
- 项目统计：按工作目录聚合 token、线程数和最近活动时间。

采用系统字体、SF Symbols、玻璃材质和语义颜色；不使用 emoji 图标。刷新不得造成明显布局跳动。

## 增强功能

在核心功能稳定后加入以下低风险增强：

- 额度速度：比较窗口已经过比例与已用比例，标记“使用偏快/正常/宽裕”。
- 阈值通知：剩余 20%、10%、5% 时分别提醒，每个窗口周期每个阈值只提醒一次。
- 重置倒计时与本地时区绝对时间。
- 登录时启动、全局快捷键和手动刷新。
- 省电策略：窗口不在前台时停止粒子效果，后台降低刷新与解析频率。

不在首版增加多账户切换、云同步、远程服务器或付费系统。

## 刷新、缓存与错误处理

- 额度默认每 60 秒刷新；收到 app-server `account/rateLimits/updated` 时合并有效字段或重新读取快照。
- token 与任务默认每 60 秒增量刷新；主窗口打开或用户点击刷新时立即更新。
- 后台缓存上一次有效快照。临时错误显示数据时间和“暂时无法刷新”，不能把旧数据伪装成实时数据。
- app-server 不可用时进入 SQLite/JSONL-only 模式：token 和任务继续工作，额度显示不可用。
- 文件不存在、数据库锁定、JSONL 截断和未知模型均应局部降级，不导致应用退出。
- 不读取或输出 `~/.codex/auth.json` 中的敏感 token。

## 安装与发布

- `make build` 生成临时签名的 `.app`。
- `make install` 安装至 `/Applications/CodexUsage.app`。
- `make release` 生成当前架构 DMG 和 SHA-256。
- 支持 Intel `x86_64` 与 Apple Silicon `arm64` 构建。
- 没有 Developer ID 时明确标注“未 notarize”，README 提供首次右键打开和系统设置放行步骤。
- GitHub 仓库为 `Ustinian-J/CodexUsage`，默认公开；首版源码和文档推送到 `main`。

## 测试与验收

必须通过：

1. 额度归类测试：5h/7d、单窗口、反转槽位、重复窗口、未知窗口和越界百分比。
2. Token 聚合测试：累计快照去重、跨日、滚动 7 天、缓存输入拆分和归档 session。
3. 任务测试：今日边界、归档状态、automation、空任务和项目进度。
4. 构建测试：`make build`、codesign 校验和 Mach-O 架构检查。
5. 本机数据探针：输出结构化 JSON，确认额度、今日/7天/累计 token 与任务均有可解释来源。
6. UI 人工检查：菜单栏圆环可见、数字不裁切、菜单可打开、刷新工作、主窗口与退出行为正常。
7. DMG 检查：可挂载、包含 `.app` 和 Applications 快捷方式、校验和匹配。

## 成功标准

用户可从 GitHub 下载或本地构建 DMG，将应用拖入 Applications，启动后无需额外 API key，即可在菜单栏看到真实的 Codex 5 小时和 7 天剩余比例；点击后可查看 token 三时间段统计、今日任务和项目进度。断网或部分本机数据缺失时应用仍可运行并诚实显示不可用或估算状态。
