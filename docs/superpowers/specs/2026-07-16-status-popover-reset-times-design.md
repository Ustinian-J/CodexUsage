# CodexUsage 小页面重置时间与入口文案设计

## 目标

优化菜单栏小页面的信息层级，消除“打开 Codex”的入口歧义，并用独立的两行卡片同时展示 5 小时与 7 天额度的下次重置时间。

## 已确认需求

- 底部左侧按钮从“打开 Codex”改为“打开主界面”；英文为“Open Main Window”。
- 按钮仍执行当前 `openCurrent` 行为，只打开 CodexUsage 主窗口，不启动 Codex 客户端。
- 从菜单栏小页面移除整块版本更新卡片 `AppUpdateMenuRow`。
- 设置页中的自动检查、手动检查、下载与忽略版本能力保持不变。
- 在“可用重置 / 最近到期”卡片上方新增同视觉体系的额度重置时间卡片。
- 新卡片固定显示两行：5h 和 7d，不将两者合并为一个“最早重置”。
- 允许增加小页面高度，并让底部“打开主界面 / 设置 / 退出”按钮整体下移，以获得更舒展的间距。

## 信息架构

Codex 视图从上到下排列为：

1. 应用标题与刷新按钮。
2. Codex / Claude Code 切换器。
3. 当前运行时额度摘要卡片。
4. 新增的额度重置时间卡片。
5. 现有的可用重置、最近到期与可选订阅到期卡片。
6. 底部主界面、设置、退出按钮。

Claude Code 继续遵守现有策略：不展示 Codex 的额度周期、重置券或订阅信息。

## 新增卡片视觉

新增卡片与现有 `accountCycleRow` 使用相同的圆角、底色、描边、水平内边距与字体层级。卡片内部使用两行等宽信息：

- 第一行：左侧“5h 下次重置”，右侧显示 5 小时额度的 `resetsAt`。
- 第二行：左侧“7d 下次重置”，右侧显示 7 天额度的 `resetsAt`。

英文文案分别为“5h next reset”和“7d next reset”。时间使用现有紧凑格式：当天显示 `HH:mm`，非当天显示 `M/d HH:mm`。单个时间缺失时该行仍保留并显示 `--`，避免布局跳动或误把另一个窗口的时间代入。

卡片内两行之间使用轻量分隔或稳定的垂直间距，不增加额外图标，避免与下面的“可用重置”含义混淆。数值使用圆角等宽数字并右对齐。

## 尺寸与布局

菜单栏小页面宽度保持 380 pt。固定高度从 412 pt 增加到 432 pt，新增空间用于卡片间距和底部按钮下移，不压缩运行时摘要卡片。若渲染验证显示 432 pt 仍导致裁切，允许只增加高度，不缩小字体、卡片或间距。

版本更新卡片移除后释放的纵向空间由新卡片与更舒展的内容间距使用。底部按钮保持三等分、32 pt 最小高度和现有卡片式按钮外观。

## 数据与错误处理

- 5h 行读取当前 Codex snapshot 的 `fiveHourQuota?.resetsAt`。
- 7d 行读取当前 Codex snapshot 的 `sevenDayQuota?.resetsAt`。
- 不新增网络请求，不读取新的本地文件，不访问凭据或 Keychain。
- 额度时间缺失时显示 `--`；不推断、不复制另一个窗口的时间。
- 重置券的“最近到期”继续读取可用明细中的未来 `expiresAt` 最小值，其含义与额度窗口 `resetsAt` 保持分离。

## 任务状态说明

本次不增加任务状态编辑能力。主界面任务看板继续为只读：

- Codex“进行中”：今天创建或活动、未归档，最近活动时间在当前时间前 2 小时以内。
- Codex“待处理”：今天创建或活动、未归档，但最近活动时间早于 2 小时阈值。
- Codex“完成”：今天归档的对话。
- “定时”：`~/.codex/automations` 中状态为 `ACTIVE` 的 automation。
- Claude Code 按任务文件原始状态映射；主界面不回写这些状态。

## 测试与验收

- 先为一个纯展示模型添加失败测试，覆盖 5h/7d 分行、各自时间来源、缺失值不串用和中英文标题。
- 实现后运行展示模型自测与现有状态项、额度、任务进度自测。
- 源码检查确认 `RuntimeStatusMenuView` 不再包含 `AppUpdateMenuRow`，底部中文文案为“打开主界面”。
- 渲染检查确认新卡片位于可用重置卡片上方，两个时间同时可见，底部按钮未裁切。
- 运行产品身份、macOS 兼容、CI 供应链和源码安全门禁。
- 干净 CI 上验证 Apple Silicon 与 Intel 构建，再核验产物哈希、DMG、签名、版本与架构后安装。
